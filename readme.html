GambleX - Online Gambling Website

GambleX is a modern online gambling platform featuring multiple games including Roulette, Slot Machine, Sports Betting, Plinko, and Mines. This project is built using pure HTML, CSS, and JavaScript, making it easy to run without any server setup or dependencies.

Note: This is a demonstration project and does not involve real money. All transactions are simulated using browser localStorage.

Features

  • User authentication system with Google sign-in option
  • Session management with 3-hour expiry
  • Deposit and withdrawal system (simulated)
  • Five fully playable games:
    • Roulette
    • Slot Machine
    • Sports Betting (Football and Cricket)
    • Plinko (similar to Stake)
    • Mines
  • Bet history for each game
  • Transaction history
  • Modern, responsive UI/UX

How to Run

Since this project uses only HTML, CSS, and JavaScript, no installation or build process is required:

  1. Simply open the index.html file in your web browser
  2. Register a new account or use the Google sign-in option
  3. Navigate to the Payments page to add funds (simulated)
  4. Enjoy the games!

Game Instructions

Roulette

Select a number or betting option, place your bet, and spin the wheel. Payouts vary based on your bet type.

Slot Machine

Set your bet amount and spin the reels. Match three identical symbols to win. Different symbols have different payout multipliers.

Sports Betting

Browse upcoming football and cricket matches, select odds to add to your bet slip, and place your bets.

Plinko

Set your bet amount, risk level, and number of balls, then drop the balls through the pins to land in multiplier buckets.

Mines

Set your bet amount and number of mines, then click tiles to reveal gems. Avoid mines to increase your multiplier and cash out before hitting a mine.

Technical Details

This project uses:

  • HTML5 for structure
  • CSS3 for styling and animations
  • Vanilla JavaScript for game logic and user management
  • localStorage for data persistence (user accounts, balances, bet history)

Limitations

As this is a client-side only application:

  • Data is stored in the browser's localStorage and will be lost if cleared
  • The Google sign-in is simulated (not a real OAuth implementation)
  • No real money is involved
